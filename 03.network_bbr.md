# network_bbr

你问的这两点其实是两个层面的东西：

* **BBR → 传输算法优化（你能控制）**
* **QoS / 带宽 → 运营商/云厂商策略（你基本控制不了）**

我分开讲清楚，并给你“工程上可执行”的判断方法。

---

# 一、BBR 是什么（你那两条命令在干嘛）

你执行的：

```bash
sysctl -w net.core.default_qdisc=fq
sysctl -w net.ipv4.tcp_congestion_control=bbr
```

本质是在改 Linux 内核的 TCP 拥塞控制算法。

---

## 🔧 背景（核心概念）

传统 TCP（如 CUBIC）：

* 通过“丢包”判断网络拥塞
* 一旦丢包 → 直接降速

👉 在“跨境链路 / 拥塞链路”下表现很差

---

## 🚀 BBR（Bottleneck Bandwidth and RTT）

是 Google 提出的算法：

👉 核心思想：

> 用带宽 + RTT 估算网络容量，而不是等丢包

---

## 📊 对比效果

| 算法        | 特点       |
| --------- | -------- |
| CUBIC（默认） | 稳，但容易降速  |
| BBR       | 更激进，吞吐更高 |

---

## 👉 那两条命令做了什么？

### 1️⃣

```bash
net.core.default_qdisc=fq
```

👉 启用公平队列（Fair Queue）
👉 让不同连接更均匀分配带宽

---

### 2️⃣

```bash
net.ipv4.tcp_congestion_control=bbr
```

👉 切换 TCP 算法为 BBR

---

# 二、是不是“执行就完事了”？

👉 **不完全**

---

## ✅ 临时生效（你现在这样做）

* 重启后会失效

---

## ✅ 正确做法（永久生效）

编辑：

```bash
/etc/sysctl.conf
```

加入：

```bash
net.core.default_qdisc=fq
net.ipv4.tcp_congestion_control=bbr
```

然后执行：

```bash
sysctl -p
```

---

## ✅ 验证是否生效

```bash
sysctl net.ipv4.tcp_congestion_control
```

输出：

```bash
bbr
```

---

# 三、BBR 有什么实际好处？

## ✅ 在你的场景（翻墙）：

* 提升跨境吞吐
* 降低“忽快忽慢”
* 提高弱网络稳定性

---

## ❗ 但要明确：

> ❗ BBR ≠ 带宽翻倍神器
> ❗ 它只能“更好利用已有带宽”

---

## 👉 举例：

| 场景           | 效果           |
| ------------ | ------------ |
| 原本 50Mbps 可用 | → 可能变 70Mbps |
| 原本被限 5Mbps   | → 还是 5Mbps ❌ |
