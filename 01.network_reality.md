# network_reality

下面给你一套**可直接落地的 Reality（XTLS）方案**，重点是：
👉 **无域名 / 不依赖 Cloudflare / 抗主动探测更强**
👉 比 WS+TLS 更“像正常流量”

我给你两种方式：

* ✅ **一键部署（推荐）**
* ✅ **手动配置（可控性更高）**
* ✅ 客户端完整参数

---

# 一、Reality 架构（你要先理解）

```id="k2z0qp"
Client
   ↓（TLS伪装）
Reality（伪装成真实网站，比如：www.microsoft.com）
   ↓
Xray（VLESS）
```

👉 核心点：

* 不需要证书
* 不需要域名
* 通过“冒充真实网站 TLS”实现混淆

# 三、手动配置（核心原理版）

如果你想完全掌控：

---

## 1️⃣ 生成密钥对

```bash id="wq1f5l"
/usr/local/bin/xray x25519
```

输出：

```id="n8x7xy"
Private key: xxx
Public key: xxx
```

---

## 2️⃣ 服务端配置

编辑：

```bash id="0qzq7u"
/usr/local/etc/xray/config.json
```

---

### 示例：

```json id="w9ht2t"
{
  "inbounds": [
    {
      "port": 443,
      "protocol": "vless",
      "settings": {
        "clients": [
          {
            "id": "你的UUID",
            "flow": "xtls-rprx-vision"
          }
        ],
        "decryption": "none"
      },
      "streamSettings": {
        "network": "tcp",
        "security": "reality",
        "realitySettings": {
          "show": false,
          "dest": "www.microsoft.com:443",
          "xver": 0,
          "serverNames": [
            "www.microsoft.com"
          ],
          "privateKey": "你的PrivateKey",
          "shortIds": [
            "abcd1234"
          ]
        }
      }
    }
  ],
  "outbounds": [
    {
      "protocol": "freedom"
    }
  ]
}
```

---

### 启动：

```bash id="i0i9ib"
systemctl restart xray
```

# 四、客户端配置（重点）

支持客户端：

* Windows：v2rayN（最新版）
* Android：v2rayNG
* iOS：Shadowrocket / Stash / sing-box

---

## 必填参数：

| 项目       | 值                |
| -------- | ---------------- |
| 地址       | VPS IP           |
| 端口       | 443              |
| UUID     | 服务端一致            |
| 加密       | none             |
| 传输       | tcp              |
| flow     | xtls-rprx-vision |
| security | reality          |

---

## Reality 特有参数：

| 项目          | 值                                             |
| ----------- | --------------------------------------------- |
| public key  | 服务端生成                                         |
| short id    | abcd1234                                      |
| server name | [www.microsoft.com](http://www.microsoft.com) |

---

## 标准 URL（可导入）

```id="h8l0mr"
vless://UUID@你的IP:443?encryption=none&flow=xtls-rprx-vision&security=reality&sni=www.microsoft.com&fp=chrome&pbk=PublicKey&sid=abcd1234&type=tcp#reality-node
```

# 五、Reality 的关键理解（非常重要）

## ✔ 优势

* 不走 WebSocket（更难识别）
* 不依赖域名/CDN
* TLS 指纹接近真实网站
* 抗主动探测强

---

## ❗ 注意点

### 1️⃣ “伪装站点”不是随便选

推荐：

* `www.microsoft.com`
* `www.apple.com`
* `www.cloudflare.com`

👉 特征：

* 大厂
* 全球 CDN
* TLS 稳定

---

### 2️⃣ IP 仍然是核心风险

Reality 只是：

> ❗ 降低识别概率
> ❗ 不是防封 IP

---

### 3️⃣ 不要滥用

* ❌ 不要高并发
* ❌ 不要跑 PT / 大下载
* ❌ 不要多人共享

---
