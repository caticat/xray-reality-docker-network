# Linode 防火墙与 SSH 端口配置

## 概述

Linode 防火墙（Cloud Firewall）在网络层控制入站/出站流量，比 VPS 内部 ufw 更优：
- **VPS 外层拦截**：未放行的流量根本不到达服务器，减少攻击面
- **独立于实例**：防火墙规则与实例解耦，换 IP/重建时规则自动生效
- **API 管理**：可通过脚本批量管理，无需逐台 SSH 配置

---

## 方案一：通过 Linode 控制台配置（推荐新手）

### 1. 创建防火墙

1. 登录 [cloud.linode.com](https://cloud.linode.com) → **Firewalls** → **Create Firewall**
2. 填写 Label（如 `reality-fw`）
3. 配置 Inbound Rules：

| Protocol | Port Range | Sources | 用途 |
|----------|-----------|---------|------|
| TCP | 22 | 0.0.0.0/0 | SSH（建议后续限制 IP） |
| TCP | 443 | 0.0.0.0/0 | Reality 代理 |

4. Outbound Rules：保持默认 **Allow All**
5. 创建后，在 **Linodes** 页面将防火墙分配给目标实例

### 2. 修改 SSH 端口

> **顺序很重要**：先在防火墙放行新端口，再修改 SSH 配置，否则会锁死自己。

```bash
# 1. 修改 sshd 配置
sudo sed -i 's/^#Port 22/Port 2222/' /etc/ssh/sshd_config
grep -q "^Port " /etc/ssh/sshd_config || echo "Port 2222" | sudo tee -a /etc/ssh/sshd_config

# 2. 覆盖 ssh.socket（Ubuntu 24.04 用 socket activation 管理端口，sshd_config 中的 Port 会被忽略）
sudo mkdir -p /etc/systemd/system/ssh.socket.d/
sudo tee /etc/systemd/system/ssh.socket.d/override.conf <<EOF
[Socket]
ListenStream=
ListenStream=0.0.0.0:2222
ListenStream=[::]:2222
EOF
# ListenStream= (空) 清除默认 22，再分别设 IPv4 和 IPv6

sudo systemctl daemon-reload
sudo systemctl restart ssh.service 或 sudo systemctl restart ssh.socket

# 3. 验证新端口生效（另开终端测试，不要关闭当前连接）
ssh -p 2222 root@your-ip
```

确认新端口能连接后，回到 Linode 控制台：
- 防火墙 Inbound Rules 中删除 TCP 22 规则
- 保留 TCP 443 和 TCP 2222

## 方案二：通过 API 配置（适合脚本化管理）

### 1. 创建防火墙

```bash
TOKEN="your_linode_api_token"
SSH_PORT=2222  # 自定义 SSH 端口

curl -X POST https://api.linode.com/v4/networking/firewalls \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"label\": \"reality-fw\",
    \"rules\": {
      \"inbound_rules\": [
        {
          \"protocol\": \"TCP\",
          \"ports\": \"$SSH_PORT\",
          \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
        },
        {
          \"protocol\": \"TCP\",
          \"ports\": \"443\",
          \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
        }
      ],
      \"outbound_rules\": [
        {
          \"protocol\": \"TCP\",
          \"ports\": \"1-65535\",
          \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
        },
        {
          \"protocol\": \"UDP\",
          \"ports\": \"1-65535\",
          \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
        }
      ]
    }
  }"
```

记录返回的 `id` 字段，即 `FIREWALL_ID`。

### 2. 创建实例时挂载防火墙

在 `linode-ctl.sh` 中设置环境变量：

```bash
export LINODE_FIREWALL_ID=14171990   # 替换为你的防火墙 ID
export LINODE_SSH_PORT=2222         # 自定义 SSH 端口

# 创建实例（自动挂载防火墙 + 配置 SSH 端口）
bash linode-ctl.sh create sg-sin-2
```

### 3. 更新已有防火墙规则

```bash
FIREWALL_ID=14171990

curl -X PUT https://api.linode.com/v4/networking/firewalls/$FIREWALL_ID/rules \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"inbound_rules\": [
      {
        \"protocol\": \"TCP\",
        \"ports\": \"2222\",
        \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
      },
      {
        \"protocol\": \"TCP\",
        \"ports\": \"443\",
        \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
      }
    ],
    \"outbound_rules\": [
      {
        \"protocol\": \"TCP\",
        \"ports\": \"1-65535\",
        \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
      },
      {
        \"protocol\": \"UDP\",
        \"ports\": \"1-65535\",
        \"addresses\": { \"ipv4\": [\"0.0.0.0/0\"] }
      }
    ]
  }"
```

## 安全建议

### SSH 端口选择

| 类型 | 示例 | 说明 |
|------|------|------|
| 避免使用 | 22 | 扫描器默认目标，日志会被爆破记录刷屏 |
| 推荐范围 | 1024-65535 | 非特权端口，不会与系统服务冲突 |
| 常见选择 | 2222, 8022, 22222 | 容易记忆，但不唯一 |
| 最佳实践 | 随机高位端口如 38917 | 降低被扫描概率 |

### 进一步加固

```bash
# 1. 禁用密码登录（仅密钥认证，最关键的一步）
sudo sed -i 's/^#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sudo systemctl restart ssh

# 2. 限制 SSH 来源 IP（如果你有固定 IP）
# 在 Linode Firewall 中将 SSH 规则的 Sources 改为你的 IP，如：
# "addresses": { "ipv4": ["1.2.3.4/32"] }

# 3. 安装 fail2ban（自动封禁爆破 IP）
sudo apt install fail2ban -y
sudo systemctl enable --now fail2ban
```

> **注意**：禁用密码登录前，确保已配置 SSH 密钥并测试连接成功，否则会永久锁死。

## 架构图

```
Internet
   │
   ▼
┌─────────────────────────────────┐
│  Linode Cloud Firewall (外层)     │
│  - TCP 2222 (SSH) → 放行         │
│  - TCP 443   (代理) → 放行       │
│  - 其他端口 → 丢弃               │
└─────────────────────────────────┘
   │
   ▼
┌─────────────────────────────────┐
│  VPS (Ubuntu 24.04)             │
│  ┌───────────────────────────┐  │
│  │ ufw (内层，可选)            │  │
│  │ - 2222/tcp                │  │
│  │ - 443/tcp                 │  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │ sshd (Port 2222)          │  │
│  │ xray (Port 443, Reality)  │  │
│  └───────────────────────────┘  │
└─────────────────────────────────┘
```

两层防火墙可共存，但功能重叠。Linode Firewall 已足够，ufw 可作为额外防线保留。
